Section 01

EXECUTIVE SUMMARY

Blueprint V4 is the production-grade synthesis of V3's event-driven agentic foundation with the PoV's missing capabilities. Eight net-new agents, a conversational UX layer, attorney/litigation early warning, regulatory rule engine, and a full governance mesh are the headline additions. V4 targets L3 AI-Centric maturity as the end-state.

V4 Thesis: V3 established the right architecture (event-driven, agentic, deterministic). V4 fills every functional gap the PoV exposed — attorney representation detection, medical record AI, conversational FNOL UX, state regulatory engine, subrogation intelligence, damage AI feedback loop — while adding industrialization controls (prompt versioning, bias monitoring, DR audit trail, multi-LLM abstraction) that make it Accenture-grade.
01
Agentic Mesh Expands from 13 → 21 Agents
V4 adds 8 net-new specialized agents: Attorney/Litigation Early Warning, Medical Record AI Reviewer, Conversational Orchestration (L3 UX), State Regulatory Compliance, Subrogation Intelligence, Adjuster Co-Pilot, Salvage & Total Loss, and Settlement Negotiation. Each carries its own event trigger, deterministic logic layer, confidence threshold, and HITL gate.
02
L3 AI-Centric UX as Primary Target
The Conversational Orchestration Agent replaces IVR/portal as the primary claimant interface at L3. The Adjuster Co-Pilot Agent replaces the Guidewire ClaimCenter desktop as the adjuster's day-to-day surface. SOR collapses to a thin data-write service. This is the architectural inversion that defines L3 maturity.

AI agents operate within a deterministic harness: SLA timers enforce acknowledgement and escalation regardless of AI confidence; business rules validate every agent output before SOR write; exception branches are fully enumerated. The V4 principle: AI proposes, deterministic logic disposes. No agent can write to SOR without passing the rule validation gate.
09
Event-Driven Parallel Execution
All V4 agents subscribe to the central Event Bus. Fraud Detection, Damage Estimation, Coverage Validation, and BI Triage execute in parallel on the same ClaimIntakeCompleted event — not sequentially. P99 FNOL completion time: <4 minutes (APD, no injury). Event schema versioned via Confluent Schema Registry; consumers declare compatibility.

The deterministic layer wraps every AI invocation. Rules, SLA timers, and exception paths are fully specified. No AI output reaches the SOR without passing the rule validation gate. Classification: ✔ Fully Automated · ⚠ Partial · ✖ Requires Human.

Section 06

TARGET ARCHITECTURE — V4

Six-layer architecture with strict separation of concerns. Agents never touch SOR directly. LLM Gateway abstracts model provider. Governance layer is cross-cutting — not a single layer but a service mesh injected at every agent invocation.

L1 · Experience
L3 Target (V4 New): Conversational Orchestration Agent (A-15) · Adjuster Co-Pilot (A-16) · Contact Center AI (A-21) · Mobile/Web FNOL App · IVR (L1/L2 transitional). At L3 the AI agent interface IS the primary UX — portal and IVR are legacy fallbacks. Single-session continuity across channels.
L2 · Agent Runtime
21 Specialized Agents deployed as stateless microservices (containers). Each agent: system prompt (versioned, hashed) · tool call schema · confidence threshold · HITL flag logic · Decision Record emitter. LLM Gateway service decouples agents from model provider — config-driven model selection. Agent SDK enforces contract.
L3 · Orchestration
Event Bus (Confluent Kafka) — schema-versioned events, 7-yr retention. Workflow Engine (Temporal / AWS Step Functions) — deterministic sequencing, SLA timers, retry + dead-letter logic. Parallel fan-out on ClaimIntakeCompleted. No agent is an orchestrator — all coordination via event subscription. Saga pattern for distributed transaction management.
L4 · Governance
Cross-cutting mesh — injected at every agent invocation: Decision Record Writer (A-18) · Bias Monitor (A-20) · Regulatory Compliance Engine (A-17) · Prompt Version Registry (git-backed hash store) · PII Vault (tokenization, consent ledger) · Model Performance Tracker (drift detection, quarterly retrain trigger). All governance artifacts immutable.
L5 · Integration
API Fabric (Kong / AWS API GW): OAuth 2.0 + mTLS all domains · circuit breakers · rate limiting · schema validation. SOR Adapters (Guidewire · Duck Creek · ALIP · Custom) — canonical FNOL Payload v4 translated to SOR format by A-14 only. Vendor connectors (telematics, fraud, damage, medical, salvage, legal data). Event-driven inbound (TSP webhooks).
L6 · Systems of Record
At L1/L2: Guidewire ClaimCenter / Duck Creek Claims / ALIP — primary claims management interface. At L3: SOR collapses to thin data-write service; adjuster interaction moves to Co-Pilot Agent surface. Policy Admin (PAS) · Billing · Document Management · Data Lake (ML training, analytics, regulatory reporting). Canonical schema ensures SOR-independence.
Multi-LLM Abstraction (V4 New): LLM Gateway service sits between Agent Runtime and all model providers. Default: claude-sonnet-4-20250514. Config-switchable to GPT-4o or Gemini Pro per agent. Enables: (1) provider failover without agent redeployment; (2) cost optimization by routing simpler tasks to cheaper models; (3) A/B testing model versions with Decision Record capture; (4) compliance with carrier model approval requirements.
